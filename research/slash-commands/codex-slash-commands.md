# Codex Slash Commands Research

Source project: `/Users/season/Personal/codex` at branch `main`, commit `3deda3116c`, clean worktree. Citations are repo-relative paths under that project.

## Executive Summary

Codex implements slash commands as a TUI-side command layer, not as prompt templates or backend API commands. The core command surface is a Rust enum, `SlashCommand`, with command names, aliases, descriptions, visibility, "supports inline args", "available during task", and "available in side conversation" metadata all attached in one place (`codex-rs/tui/src/slash_command.rs:7`, `codex-rs/tui/src/slash_command.rs:12`, `codex-rs/tui/src/slash_command.rs:78`, `codex-rs/tui/src/slash_command.rs:146`, `codex-rs/tui/src/slash_command.rs:166`, `codex-rs/tui/src/slash_command.rs:179`).

The runtime has one dynamic extension to that static enum: model service tiers are exposed as slash commands when Fast Mode is enabled. Those are represented as `ServiceTierCommand` and wrapped next to built-ins in `SlashCommandItem`; they are inserted immediately after `/model` in the suggestion list (`codex-rs/tui/src/bottom_pane/slash_commands.rs:13`, `codex-rs/tui/src/bottom_pane/slash_commands.rs:20`, `codex-rs/tui/src/bottom_pane/slash_commands.rs:86`, `codex-rs/tui/src/bottom_pane/slash_commands.rs:94`, `codex-rs/tui/src/chatwidget/service_tiers.rs:75`, `codex-rs/tui/src/chatwidget/service_tiers.rs:82`).

There is no file-backed or user-defined slash-command registry in the TUI path that I found. Search points all real dispatch through `SlashCommand`, `ServiceTierCommand`, `ChatComposer`, and `ChatWidget`; `docs/slash_commands.md` is only a pointer to external product docs (`docs/slash_commands.md:1`, `docs/slash_commands.md:3`).

## Command Registry

Built-ins live in `SlashCommand`. Strum derives parse and display behavior, and `serialize_all = "kebab-case"` makes enum variants become command names by default (`codex-rs/tui/src/slash_command.rs:8`, `codex-rs/tui/src/slash_command.rs:11`). Aliases are encoded as enum attributes: `/clean` parses as `Stop`, `/pet` parses as `Pets`, and `/approve` is the public name for `AutoReview` (`codex-rs/tui/src/slash_command.rs:25`, `codex-rs/tui/src/slash_command.rs:51`, `codex-rs/tui/src/slash_command.rs:62`, `codex-rs/tui/src/slash_command.rs:260`).

The enum order is presentation order in the popup, with a warning not to alphabetize it (`codex-rs/tui/src/slash_command.rs:13`). `built_in_slash_commands()` iterates that enum, filters invisible commands, and returns `(name, command)` pairs (`codex-rs/tui/src/slash_command.rs:235`, `codex-rs/tui/src/slash_command.rs:245`). Visibility is platform/build dependent for a few commands, such as Windows-only `/sandbox-add-read-dir`, non-Android `/copy`, and debug-only `/rollout` and `/test-approval` (`codex-rs/tui/src/slash_command.rs:235`, `codex-rs/tui/src/slash_command.rs:237`).

Feature and context gating are centralized in `BuiltinCommandFlags`. `builtins_for_input()` filters commands by feature flags for collaboration modes, apps/connectors, plugins, goals, personality, realtime/audio, elevated sandbox, and active side conversation (`codex-rs/tui/src/bottom_pane/slash_commands.rs:56`, `codex-rs/tui/src/bottom_pane/slash_commands.rs:70`). `find_builtin_command()` intentionally ignores side-conversation gating for exact lookup so dispatch can show a specific "unavailable in side conversations" error instead of treating the command as unknown (`codex-rs/tui/src/bottom_pane/slash_commands.rs:109`, `codex-rs/tui/src/bottom_pane/slash_commands.rs:113`).

## Parsing And Validation

Parsing is deliberately small. `parse_slash_name()` accepts only text that starts with `/`, then takes the first non-whitespace token as the command name and returns the rest plus a byte offset into the original line/text (`codex-rs/tui/src/bottom_pane/prompt_args.rs:1`, `codex-rs/tui/src/bottom_pane/prompt_args.rs:7`). Empty command names fail.

`SlashInput` wraps that parser with composer state: slash commands must be enabled, not in bash mode, and not intentionally escaped with a leading space (`codex-rs/tui/src/bottom_pane/chat_composer/slash_input.rs:46`, `codex-rs/tui/src/bottom_pane/chat_composer/slash_input.rs:68`, `codex-rs/tui/src/bottom_pane/chat_composer/slash_input.rs:107`). Command names containing another slash are treated as ordinary text, which keeps strings like paths from becoming command errors (`codex-rs/tui/src/bottom_pane/chat_composer/slash_input.rs:79`, `codex-rs/tui/src/bottom_pane/chat_composer/slash_input.rs:112`).

Bare commands and inline commands are separate paths. `bare_command()` only dispatches when the first line is `/name` with no argument text. `inline_command()` only dispatches when the command supports inline args and has non-empty rest text (`codex-rs/tui/src/bottom_pane/chat_composer/slash_input.rs:89`, `codex-rs/tui/src/bottom_pane/chat_composer/slash_input.rs:107`, `codex-rs/tui/src/slash_command.rs:146`).

Unknown slash commands are errors in the immediate TUI submission path. `prepare_submission_text_with_options()` validates before normal text submission, emits "Unrecognized command '/name'. Type "/" for a list of supported commands.", restores the draft, and returns `None` (`codex-rs/tui/src/bottom_pane/chat_composer.rs:2640`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:2669`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:2674`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:2680`). Queued slash text defers validation until dequeue (`codex-rs/tui/src/bottom_pane/chat_composer.rs:2757`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:2762`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:801`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:831`).

## Popup And Completion

`ChatComposer` owns key routing and popup synchronization. Its top-level doc spells out that it routes keys to slash/file/skill/app mention popups and stages slash commands for local history before `ChatWidget` dispatches them (`codex-rs/tui/src/bottom_pane/chat_composer.rs:1`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:5`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:35`). Slash command parsing can be disabled for embedded plain-text composer surfaces (`codex-rs/tui/src/bottom_pane/chat_composer.rs:310`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:331`).

The slash popup appears only when the cursor is editing the initial `/name` token on the first line, slash commands are enabled, bash mode is off, and file/mention popups are not active (`codex-rs/tui/src/bottom_pane/chat_composer.rs:3477`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:3516`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:3572`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:3586`). If the user is inside an `@token`, file search wins so `/review @docs/...` can still complete file paths (`codex-rs/tui/src/bottom_pane/chat_composer.rs:3594`).

`CommandPopup` is a view over the same filtered command list. It hides debug commands and `/apps` from the popup, hides alias commands such as `/quit` and `/btw` in the empty default list, but still shows aliases when the user types their prefixes (`codex-rs/tui/src/bottom_pane/command_popup.rs:20`, `codex-rs/tui/src/bottom_pane/command_popup.rs:73`, `codex-rs/tui/src/bottom_pane/command_popup.rs:81`, `codex-rs/tui/src/bottom_pane/command_popup.rs:143`). Filtering is exact/prefix over the token after the first `/`, preserving registry presentation order (`codex-rs/tui/src/bottom_pane/command_popup.rs:95`, `codex-rs/tui/src/bottom_pane/command_popup.rs:143`, `codex-rs/tui/src/bottom_pane/command_popup.rs:156`).

Popup keys are handled in `slash_input.rs`: Up/Down or Ctrl+P/Ctrl+N move selection, Esc dismisses, Tab completes or dispatches special commands, `/` accepts the highlighted command as text completion, and Enter dispatches the selected item (`codex-rs/tui/src/bottom_pane/chat_composer/slash_input.rs:209`, `codex-rs/tui/src/bottom_pane/chat_composer/slash_input.rs:231`, `codex-rs/tui/src/bottom_pane/chat_composer/slash_input.rs:255`, `codex-rs/tui/src/bottom_pane/chat_composer/slash_input.rs:262`, `codex-rs/tui/src/bottom_pane/chat_composer/slash_input.rs:310`, `codex-rs/tui/src/bottom_pane/chat_composer/slash_input.rs:348`). `/skills` is currently special-cased to dispatch immediately on Tab (`codex-rs/tui/src/bottom_pane/chat_composer/slash_input.rs:489`).

## Submission And Queueing

The composer returns explicit `InputResult` variants: normal submitted text, queued text with an action, bare built-in command, bare service-tier command, inline command with args, or none (`codex-rs/tui/src/bottom_pane/chat_composer.rs:271`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:273`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:287`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:289`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:295`). `ChatWidget` then maps those variants to app-level effects (`codex-rs/tui/src/chatwidget/input_flow.rs:10`, `codex-rs/tui/src/chatwidget/input_flow.rs:54`).

On live submission, the composer tries bare slash dispatch first, then inline slash dispatch, then normal text submission (`codex-rs/tui/src/bottom_pane/chat_composer.rs:2784`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:2791`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:2835`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:2839`). Bare dispatch clears the textarea and returns either `InputResult::Command` or `InputResult::ServiceTierCommand` (`codex-rs/tui/src/bottom_pane/chat_composer.rs:2875`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:2886`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:2889`). Inline dispatch rebases text elements into the args string so pasted placeholders, mentions, and other rich elements survive argument extraction (`codex-rs/tui/src/bottom_pane/chat_composer.rs:2895`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:2909`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:2926`).

When input is queued during a running task, `QueuedInputAction` records whether the queued text should later be plain text, slash-parsed, or run as a user shell command (`codex-rs/tui/src/bottom_pane/chat_composer.rs:299`, `codex-rs/tui/src/bottom_pane/chat_composer/slash_input.rs:196`). On dequeue, `submit_queued_slash_prompt()` re-runs slash parsing, exact command lookup, side/task availability checks, inline-arg handling, and then returns a `QueueDrain` decision so quick local commands can drain through while modal/turn-starting commands stop the queue (`codex-rs/tui/src/chatwidget/slash_dispatch.rs:801`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:831`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:844`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:857`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:928`).

## Dispatch

`ChatWidget::dispatch_command()` is the main built-in command switch. Before the switch, it blocks commands unavailable in side conversations, blocks `/side` during review, and blocks commands whose `available_during_task()` metadata says they cannot run while a task is active (`codex-rs/tui/src/chatwidget/slash_dispatch.rs:132`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:133`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:136`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:139`).

The switch mixes direct UI state changes, app events, and backend ops: `/new` sends `AppEvent::NewSession`, `/clear` sends `ClearUi`, `/init` submits a bundled prompt after checking for an existing `AGENTS.md`, `/compact` sends compact, `/review` opens review UI, `/diff` spawns a workspace diff task, `/mcp` renders MCP output, and `/quit`/`/exit` request shutdown (`codex-rs/tui/src/chatwidget/slash_dispatch.rs:150`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:164`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:176`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:188`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:195`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:338`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:418`).

Inline args are handled in a second switch. Supported examples include `/ide <args>`, `/mcp verbose`, `/keymap debug`, `/raw on|off`, `/rename <name>`, `/plan <prompt>`, `/goal <objective|clear|edit|pause|resume>`, `/side <prompt>`, `/review <instructions>`, `/resume <id-or-name>`, `/sandbox-add-read-dir <path>`, and `/pets <id|off>` (`codex-rs/tui/src/chatwidget/slash_dispatch.rs:474`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:587`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:591`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:595`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:609`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:620`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:632`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:653`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:752`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:770`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:775`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:779`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:783`).

Service-tier commands are dispatched outside the built-in switch. If not in a side conversation, they toggle the active service tier and persist it through both a turn-context override and config update (`codex-rs/tui/src/chatwidget/slash_dispatch.rs:53`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:63`, `codex-rs/tui/src/chatwidget/service_tiers.rs:66`, `codex-rs/tui/src/chatwidget/service_tiers.rs:106`).

## Local Recall And Tests

Slash command history is intentionally two-phase. The composer stages the command text before clearing the input, then `ChatWidget` records that staged entry after dispatch (`codex-rs/tui/src/bottom_pane/chat_composer.rs:35`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:356`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:1483`, `codex-rs/tui/src/chatwidget/slash_dispatch.rs:40`). Popup-selected partial input records the canonical command, so completing `/di` to `/diff` recalls `/diff`, not `/di` (`codex-rs/tui/src/bottom_pane/chat_composer.rs:2975`).

`/clear` is excluded from recall, unknown slash commands are not recorded, and unavailable known commands are recorded so the user can edit/retry them (`codex-rs/tui/src/bottom_pane/chat_composer.rs:2968`, `codex-rs/tui/src/bottom_pane/chat_composer.rs:2979`, `codex-rs/tui/src/chatwidget/tests/slash_commands.rs:1139`, `codex-rs/tui/src/chatwidget/tests/slash_commands.rs:1160`). There are focused tests for queued slash commands, queued unknown slash errors, service-tier commands, local recall, inline arg extraction, and side-conversation availability (`codex-rs/tui/src/chatwidget/tests/slash_commands.rs:90`, `codex-rs/tui/src/chatwidget/tests/slash_commands.rs:147`, `codex-rs/tui/src/chatwidget/tests/slash_commands.rs:524`).

## Implications For Nav

1. Keep the command registry boring and typed. Codex gets a lot of leverage from a central enum plus metadata methods. For nav, a TypeScript discriminated union or const registry could carry `name`, `aliases`, `description`, `supportsArgs`, `availableDuringTask`, and `availableInSideThread`.
2. Split suggestion, parsing, and dispatch. Codex's popup can be changed without touching `ChatWidget::dispatch_command()`, and dispatch can reject commands with better context-specific messages than the autocomplete layer can know.
3. Treat queued slash commands as structured queued actions, not as raw user text. Codex's `QueuedInputAction::ParseSlash` avoids accidentally sending `/compact` to the model after a running turn completes.
4. Preserve drafts on unknown commands. Codex restores the original input when validation fails, which is kinder than clearing a mistyped command.
5. Be deliberate about dynamic commands. Codex only has one dynamic category, service tiers, and it routes them through a separate `SlashCommandItem` variant rather than pretending they are built-in enum values.
6. Decide early whether nav wants user-defined prompt commands. Codex's TUI currently does not expose Pi-style prompt templates or skill slash aliases; adding that later would be a different subsystem from the hard-coded TUI command path.
