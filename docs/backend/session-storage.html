Session storage

Planned model

Session persistence is not implemented yet. The intended direction is one canonical transcript in SQLite, not one transcript per provider.

Provider adapters should encode canonical turns into the request shape needed by the current API, then decode provider output back into canonical events and turns.

Boundary rules

  • sessions identify the workspace and ordered history.
  • runs group one user request and the backend work that follows.
  • turns hold canonical user, assistant, tool, and system content.
  • provider_state can cache provider handles such as OpenAI previous_response_id, but it is not the source of truth.

Changing models mid-session should change settings only. Stored history should not become provider-shaped data.
