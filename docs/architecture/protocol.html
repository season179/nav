Protocol.

Two channels

Commands go in via JSON-RPC. Events stream out via SSE. The frontend never polls.

JSON-RPC methods

Frontend commands hit POST /rpc. Responses acknowledge state immediately.

  • initialize — handshake
  • session.create — start a session
  • session.sendMessage — user input
  • run.cancel — abort a running agent loop
  • tool.approve / tool.reject — tool permission
  • session.close — end a session

SSE event types

Events stream to GET /sessions/{id}/events. Reconnect with Last-Event-ID.

  • session.created
  • run.started / run.completed / run.failed
  • message.delta / message.completed
  • tool.call_requested / tool.approval_requested
  • tool.call_started / tool.call_completed
  • file.changed

Transport

Local HTTP on 127.0.0.1, random port, random local auth token. Frontend launchers may spawn the backend and discover its URL/token through a small bootstrap mechanism.

All protocol IDs are canonical lowercase UUIDv7 strings — they're not secrets, but they do leak approximate creation time, so authentication uses separate random tokens.
