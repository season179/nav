Who owns what

flowchart LR subgraph frontend["Frontend owns"] direction TB f1["Layout & rendering"] f2["Input handling"] f3["Reconnect UX"] f4["Optimistic UI state"] end subgraph protocol["Protocol"] p["UI-agnostic\nJSON-RPC + SSE"] end subgraph backend["Backend owns"] direction TB b1["Sessions & runs"] b2["Agent loop"] b3["Model orchestration"] b4["Filesystem & shell"] b5["Permissions & policy"] end frontend <-->|"protocol"| backend

Backend owns agent state and side effects: sessions, runs, tools, models, filesystem/shell, policy.

Frontends own presentation: layout, input, reconnect UX, optimistic UI. The protocol stays UI-agnostic.

Rule of thumb: if a concept would mean something in a CLI, an Electron app, and a web browser, it belongs in the backend. If it only makes sense on a screen, it belongs in the frontend.

Prototype gap

Today the TUI still talks to the backend over stdio with a hello check; HTTP + SSE is the target. The next milestone replaces the stdio path with the full protocol.
