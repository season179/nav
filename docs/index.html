A learning project for building a coding agent in the open: Rust backend, Go terminal UI, local JSON-RPC, and typed SSE events.

Project shape

nav keeps terminal rendering in Go/Bubble Tea and agent state in Rust. The local JSON-RPC + SSE path exists now; the fuller provider and tool loops are still being filled in.

Working rules

Learn by building

The point is to understand agent internals, not to pretend this is ready for production use.

Clear boundaries

Backend owns state & side effects. Frontend owns presentation. Protocol stays UI-agnostic.

Plain names

Prefer sessions/ over abstractions. Code should explain itself.

Replaceable frontends

Go TUI ships first, but the protocol must also support Electron and web frontends.

Folders before crates

Promote a folder to a crate only when its API is stable. Reshape while learning.
