A coding agent built from scratch — so the agent loop, tools, permissions, context, and terminal UI stay readable.

The big picture

nav is a Rust backend that runs an agent loop — calling LLMs, executing tools, enforcing guardrails — with a Go terminal UI that talks to it over HTTP.

Design principles

Every decision in nav traces back to one of these.

Learn by building

The primary purpose is understanding how coding agents work — not outperforming them.

Clear boundaries

Backend owns state & side effects. Frontend owns presentation. Protocol stays UI-agnostic.

Plain names

Prefer sessions/ over abstractions. Code should explain itself.

Replaceable frontends

Go TUI ships first, but the protocol must also support Electron and web frontends.

Folders before crates

Promote a folder to a crate only when its API is stable. Reshape while learning.
