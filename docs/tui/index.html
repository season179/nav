The TUI is a Bubble Tea program with thin entrypoints and clear internal boundaries.

flowchart TB subgraph cmd["Entrypoints"] nav["cmd/nav\nreleased binary"] navd["cmd/navd\nlocal dev"] end subgraph internal["internal/"] app["app/\ncomposition root"] client["client/\nbackend API"] ui["ui/\nviews & styles"] localdev["localdev/\nbuild & run"] end nav --> app navd --> app app --> client app --> ui app --> localdev

internal/app wires the backend client into the Bubble Tea program. internal/client speaks JSON-RPC and consumes SSE. internal/ui maps backend events to views.

nav is the released command; navd runs local builds from a checkout. Entrypoints stay thin; they delegate to internal/ immediately.

For visual direction, keep comparing the terminal shape against the sibling Crush TUI instead of inventing a new shell from docs alone.
