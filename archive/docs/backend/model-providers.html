nav/backend/model providers

Model providers

Settings file

nav-backend serve-http reads ~/.nav/settings.json by default. Set NAV_MODEL_SETTINGS to point at another file.

The shape follows Pi, the local agent runner this project borrows settings conventions from: a defaultModel plus keyed providers. Each provider names a baseUrl, api, API-key source, compatibility overrides, and available models.

{
  "defaultModel": {
    "provider": "local-gateway",
    "model": "local-coder:7b"
  },
  "providers": {
    "local-gateway": {
      "baseUrl": "http://localhost:11434/v1",
      "api": "openai-completions",
      "apiKey": { "envVar": "LOCAL_GATEWAY_API_KEY" },
      "models": [{ "id": "local-coder:7b" }]
    }
  }
}

Runtime overrides

Use NAV_MODEL_PROVIDER and NAV_MODEL together to override the default model without editing the settings file. Supplying only one is a configuration error.

Prefer environment variables for API keys. Inline keys are accepted for local testing, but they must never appear in logs, errors, fixtures, or docs examples with real values.

Concrete examples live in crates/nav-harness/examples/model-settings-*.json.
